## Design Context

### Users
SEO freelancers and agencies. Context: active audit sessions, often under time pressure, reviewing a site's health before a client call or during a sprint. The job is to **find problems fast** — not explore leisurely.

### Brand Personality
Modern · Fast · Professional

Reference products: Ahrefs (dense, data-heavy, no decoration for decoration's sake), SEMrush (comprehensive, clearly hierarchical, respects the user's intelligence).
Anti-reference: anything that looks "agency portfolio" — overly decorative, motion-heavy, soft pastel.

### Aesthetic Direction
Dark-first (already established). Green accent (`#00ff88`) as the primary success/brand signal. Blue (`#4d8dff`) for interactive/primary actions. Dense information layout is correct — don't over-space. Professional means the UI gets out of the way of the data.

Existing fonts (Manrope + Syne) are established and should be kept.

### Design Principles
1. **Data over decoration** — if a visual element doesn't help the user find a problem faster, remove it.
2. **Instant feedback** — every interaction should feel immediate. Loading states, hover states, and transitions signal responsiveness.
3. **Hierarchy through typography, not color** — use size and weight to establish hierarchy before reaching for color.
4. **Accessibility is professional** — focus states, keyboard nav, and contrast ratios are not optional.
5. **Consistency compounds** — same padding scale, same border-radius, same token usage everywhere. Drift creates cognitive noise.
